## React组件

props + state -> view

外部传的属性加内部维护的状态决定view

单向数据绑定

创建组件遵循的原则：

1、何时创建 -> 单一职责原则：每个组件只做一件事；如果组件变得复杂，应该拆分成小组件；

2、数据状态管理 -> DRY原则：能计算得到的状态不要单独存储；组件尽量无状态，所需数据通过props获取；
